# Contributing

We welcome contributions to improve examples, add routes, or enhance documentation.

## How to Contribute

1. **Fork** the repository
2. **Create a branch**: `git checkout -b my-feature`
3. **Make changes** - add new routing examples or improve docs
4. **Commit your work**: `git commit -m "Describe your changes"`
5. **Push to your fork**: `git push origin my-feature`
6. **Open a pull request** against the main repository

## Guidelines

- Keep examples simple and educational
- Follow existing code style and structure
- Update documentation when adding new routing patterns
- Test your changes with `npm run dev`

## Ideas for Contributions

- Add more routing examples
- Improve documentation with diagrams
- Add TypeScript types to existing examples
- Create interactive demos
- Translate documentation

## Questions?

Open an issue if you have questions or suggestions.
